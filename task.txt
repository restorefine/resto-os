DESIGN SYSTEM
- Color palette:
    Primary background: #0A0A0A (near black)
    Secondary background: #111111
    Card background: #1A1A1A
    Border: #2A2A2A
    Primary accent: #E8FF47 (electric yellow-green)
    Secondary accent: #FFFFFF
    Muted text: #888888
    Success: #22C55E
    Warning: #F59E0B
    Danger: #EF4444

- Typography:
    Display font: Syne (headings, nav labels, card titles)
    Body font: DM Sans (paragraphs, table data, form labels)
    Mono font: JetBrains Mono (IDs, amounts, codes)

- Design direction:
    Dark, minimal, sharp — like a premium fintech dashboard
    No rounded corners above 8px
    Subtle borders not shadows
    Yellow-green accent used sparingly for CTAs and active states
    Sidebar is dark with icon + label navigation
    Cards have a thin border, no drop shadow
    Tables are dense with hover states
    Status badges use filled pill style

AUTHENTICATION
- JWT stored in httpOnly cookie (set by Go backend)
- Three roles: admin, staff, client
- middleware.ts reads cookie, redirects to /login if missing
- Admin sees everything
- Staff sees only assigned clients
- Client sees only their portal (separate subdomain layout)
- On first login, client is forced to change password

API INTEGRATION
- Base URL from NEXT_PUBLIC_API_URL env variable
- All requests via Axios instance in lib/api.ts
- Auth token sent automatically via cookie (withCredentials: true)
- TanStack Query handles caching, refetching, loading states
- On 401 response → clear auth state → redirect to /login
- On network error → show toast notification

PAGES TO BUILD

1. LOGIN PAGE (/login)
   - Email + password form
   - Zod validation
   - On success → redirect to /dashboard
   - Error states for wrong credentials
   - RestoRefine Studios branding

2. LIVE DASHBOARD (/)
   - MRR card (total monthly recurring revenue)
   - Unpaid invoices card (count + total value)
   - Pipeline value card (total value of active leads)

FORMS
- All forms use React Hook Form + Zod
- Validation on submit and on blur
- Error messages shown inline below each field
- Loading state on submit button
- Success → close form + invalidate TanStack Query cache
- Error → show toast with error message

STATE MANAGEMENT
- Zustand store holds: currentUser, sidebarOpen, notifications[]
- TanStack Query holds all server data
- No Redux — too heavy for this project
- Optimistic updates on mark paid, approve video, move lead

NOTIFICATIONS
- Toast notifications (top right) for all actions
- Success, error, warning variants
- Auto-dismiss after 4 seconds
- Real-time notifications via SSE:
  GET /api/stream → EventSource in browser
  Events: invoice_paid, video_approved, lead_moved, portal_activated

ERROR HANDLING
- 401 → logout + redirect to login
- 403 → show "Access denied" message
- 404 → show not found state in page
- 500 → show error toast + log to console
- Network error → show "Connection lost" banner
